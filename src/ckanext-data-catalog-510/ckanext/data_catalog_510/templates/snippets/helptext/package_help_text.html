Create Dataset

There are two important entities in this catalog.

  1. the dataset
  2. the resource
The dataset includes all important (background) information about the dataset. The resource has information (location, amount, type, etc.) about the specific files related to it. A resource is always part of a dataset and one dataset can have multiple resources.

Fill in all fields to add a new dataset to the 510 data catalog. Additional information about the fields can be found below.

Dataset name

{% trans %} For the Dataset name you should choose something that is understandable for people searching for data and is short and consise.
There are no rules on what needs to be in it. Just remember that many of the details will be in the other metadata fields so they don't all need to be included in the name.

The dataset name is stored as lower case and a whitespace is replaced by a dash '-'. {% endtrans %}

Description

You are free to put in anything that is important to know about this dataset that is not recorded in any of the other fields. You can for example briefly explain how the dataset was generated or acquired.

Tags

Here you add tags. Start typing and tags that are used before are listed down so you can select them. If a tag hasn't been used before it will be newly created. Tags are used to easily filter datasets.

License

Select the license of the dataset if you know it. Otherwise leave the field empty.

Organization

Always select just 510, as is the default. So no need to change this.

Source

Put here the source of the dataset. If it is downloaded from a website you can put the url, otherwise mention the organization or person.

Dataset Owner

This is you by default. Only change it when you are adding data for someone else. Please keep the format the same as the account name, which is normally the same as the microsoft account name. E.g. "Ottow, Bouke".

Dataset Owner Email

Also filled in automatically. See Dataset Owner, only change when the owner is someone else. This email address is used for correspondence about this dataset.

Dataset Version

Mention here the version number or name when a dataset has versioning. It can also be a year or date for example for admin boundaries.

Security classification

The security classification is used for accessibility of the datasets through this catalog. Low means everyone can see the dataset in the catalog (also when not logged in). Normal means only logged in people can see the dataset and if possible a preview will be displayed. High means only logged in people can see the dataset and no preview will be displayed.

Country

Select the country which is covered by the dataset. You can select multiple countries if needed. You can also select Global. Please only select countries from the list (just start typing and select the right one).

Used in Project

Select here the name (as in Forecast) of the project where you initially used the data in. Start typing and the options will appear.

Used in Product

Select here the name (as in Forecast) of the product where you initially used the data in. Start typing and the options will appear. You can also leave it empty if it isn't used by any of the 510 products.

Expected Update Frequency

How often is the dataset expected to be updated? Is it daily, or yearly or not at all?

Accuracy

This field relates to the data quality as explained in the data responsibility policy.

Accuracy is the degree of closeness of data values to real values.

Consistency

This field relates to the data quality as explained in the data responsibility policy.

Consistency is the degree to which data values of two sets of attributes within a record, or within a dataset, or between datasets, or within a record at different points in time, comply with a rule.

Completeness

This field relates to the data quality as explained in the data responsibility policy.

Completeness is the degree to which all required data values are present.

Date added

This is filled in automatically, there is no need to change it.
