Add new resource

To add a new resource to an existing or newly created dataset you first have to select where the data(files) are stored. This is either through

  • External Link, when the data is not in the 510 data platform but can be accessed online through a link (e.g. a google sheet);
  • Databases, when the data is stored in one of the data bases in the 510 data platform;
  • Datalake, when the data is stored in the Azure Data Lake Storage in the 510 data platform.

URL (External Link)

Put here the url through which the data can be accessed.

Select database type (Databases)

Select here the type and server in the 510 data platform where the data is stored. This is either mySQL, Azure SQL or PostgreSQL.

Select database (Databases)

Select here the database (in the before selected server) where the data is stored. A popup will open that asks you to login to check if you actually have access to the database. The format is <username>@<servername>, e.g. bottow@510-postgresql-server.

Select Schema (Databases)

Select here the schema (in the before selected database) where the data is stored.

Select Table (Databases)

Select here the table (in the before selected schema) which has the data.

Select Data from Datalake (Datalake)

Click the button and browse through the file structure in the data lake to select your data. Double click to go deeper in to a container or directory and click once to select the file or directory.

Geo-data (Databases & Datalake)

Check the Is Data Geo-data? checkbox to be able to (auto-) fill in the spatial metadata fields.

If the data is a table in a postgres database the box is automatically checked if the table has a geometry column and the fields are automatically populated. Depending if the data is raster data or vector data not all the fields might be applicable.

If the data is stored in the data lake you have to select the checkbox if it is geo data. You then have to select the file from which the system has to extract the metadata from (in case you have selected a folder). And then click Auto-fill Spatial Metadata. Be warned that the file needs to be downloaded to the server which has a limit on the size.

Name

Fill in the name of the resource.

Description

Give a description to the resource. This is especially important when there are multiple resources as part of one dataset.

Format

Sometimes this is filled in automatically. If not, fill in what kind of file it is. (e.g. csv, json, etc.)
